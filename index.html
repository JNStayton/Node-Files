Open this directory in the terminal, and run the following bolded commands:

    Part 1: step1.js can read the contents of a local path, or return an error message if nonexistent.

  • 'node step1.js one.txt' - This should return the contents of one.txt
  • 'node step1.txt blah.txt' - This should return an error message

    • Part 2: step2.js can differentiate between a URL and local path, or return an error message is the path or URL does not exist

  • 'node step2.js one.txt' - This should return the contents of one.txt ("This is file one."
  • 'node step2.js http://google.com' - This should return the HTML for google.com
  • 'node step2.js http://blah.com/i-am-fake' - This should return an error message

    • Part 3: step3.js can do both step1 and step2, but can also accept '--out' as the first command line argument to optionally write over one path with a second given path. The command line argument should look like this: '--out output-filename.txt readfile-or-url'

  • 'node step3.js one.txt' - This should return the contents of one.txt
  • 'node step2.js http://google.com' - This should return the HTML for google.com
  • 'node step3.js --out new.txt one.txt' - new.txt now contains the contents of one.txt
  • 'node step3.js --out new.txt http://google.com' - new.txt now contains the HTML for google.com
  • 'node step3.js --out /i/do/not/exist.txt one.txt' - This should return an error message
